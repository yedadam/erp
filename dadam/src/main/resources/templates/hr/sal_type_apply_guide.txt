# sal1~sal10(급여유형/패키지) 급여관리 페이지 적용 가이드

---

## 1. 급여유형(패키지) 관리 UI/기능 추가
- 관리자(인사팀 등)가 sal1~sal10 각각에 대해 “포함할 수당/공제 항목”을 선택/조합할 수 있는 UI를 제공
- 예시: sal1 = 기본급, 식대, 자격수당, 소득세, 4대보험 / sal2 = 기본급, 직책수당, 복지포인트, 소득세, 4대보험, 노조비 등
- 급여유형 테이블(salary_type 등) 설계: type_code(sal1, sal2, ...), type_name, allow_codes(포함항목)

## 2. 사원별 급여유형 지정
- 사원정보(EMPLOYEES 등)에 “급여유형(sal1~sal10)”을 선택하는 필드 추가
- 사원 등록/수정 시 급여유형을 선택(드롭다운 등)

## 3. 급여명세서(급여관리) 페이지 적용
- 사원별로 “적용 급여유형(sal1, sal2, ...)”을 확인
- 해당 급여유형에 포함된 수당/공제 항목만 합산/공제하여 급여를 자동 계산해서 표시
- 급여계산 버튼 클릭 시 사원별로 지정된 급여유형(sal1, sal2, ...)을 조회, 해당 급여유형에 포함된 항목만 합산/공제하여 급여명세서 생성
- 급여명세서 PDF/엑셀/출력에도 급여유형별로 계산된 급여명세서를 출력/다운로드

## 4. 실제 적용 예시 (화면/로직)
- [급여유형 관리] : 급여유형코드, 급여유형명, 포함항목(수당/공제) 관리 UI
- [사원정보] : 사원별로 급여유형(sal1, sal2, ...) 선택
- [급여명세서] : 사원별로 지정된 급여유형(sal1, sal2, ...)에 따라 해당 급여항목만 합산/공제하여 급여 자동 계산

## 5. 실제 코드/DB 연동 예시
- 급여유형 관리 테이블: salary_type(type_code, type_name, allow_codes, com_id, ...)
- 사원정보 테이블: employees(emp_id, ..., salary_type_code, ...)
- 급여명세서 생성 로직: 사원별 salary_type_code → salary_type.allow_codes → 해당 항목만 합산/공제

## 6. 결론/실무 팁
- 급여유형(패키지) 관리 UI/DB/로직을 추가하면 sal1~sal10의 실무적 의미를 100% 반영할 수 있음
- 기존 급여항목 관리/급여명세서 구조와도 완벽하게 호환됨
