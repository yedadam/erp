# sal1~sal10(급여유형/패키지) 실무적 의미와 ERP 적용 가이드

---

## 1. sal1~sal10의 진짜 의미
- sal1, sal2, ...는 "회사가 직접 정의한 급여 계산식(=급여유형, 급여패키지, 급여그룹)"을 의미함
- 각 sal1~sal10은 '급여명세서의 한 줄'이 아니라, '급여 계산 방식(패키지/그룹/유형)' 자체를 의미

### 예시
- sal1: 기본급 + 식대 + 자격수당 - 소득세 - 4대보험 (정규직 일반 사원용)
- sal2: 기본급 + 직책수당 + 복지포인트 - 소득세 - 4대보험 - 노조비 (관리자/임원용)
- sal3: 기본급 + 상여금 + 특별성과급 - 소득세 - 4대보험 - 기타공제 (성과급 지급 대상자용)

즉, 각 sal1~sal10은 "급여 계산식(조합)"을 회사가 직접 정의해서 사원별로 어떤 급여유형(sal1, sal2, ...)을 적용할지 선택하는 구조임

---

## 2. 실무적 구조
- salaryitem: 각 수당/공제 항목(식대, 자격수당, 소득세, 4대보험 등)을 정의
- sal1~sal10: "급여 계산식(조합)"을 정의 (예: sal1 = [기본급, 식대, 자격수당, 소득세, 4대보험], sal2 = ...)
- 사원정보(EMPLOYEES 등): 각 사원별로 "적용 급여유형(sal1, sal2, ...)"을 지정

---

## 3. 급여관리 페이지에서 해야 할 일
1. 급여유형(=sal1~sal10) 관리/설정 기능 제공
   - sal1~sal10 각각에 대해 "포함할 수당/공제 항목"을 선택/조합할 수 있도록 UI 제공
2. 사원별로 적용 급여유형(sal1, sal2, ...)을 선택/지정
3. 급여명세서/계산 시 사원별로 지정된 급여유형(sal1, sal2, ...)에 따라 해당 급여항목(수당/공제)만 합산/공제하여 급여를 자동 계산

---

## 4. 결론
- sal1~sal10은 "급여항목"이 아니라, "급여 계산식(유형/패키지/그룹)"이다!
- 각 sal1~sal10에 "포함할 수당/공제 항목"을 회사가 직접 조합해서 정의
- 사원별로 적용 급여유형(sal1, sal2, ...)을 선택
- 급여관리 페이지에서는
  - sal1~sal10의 "구성(포함 항목)"을 관리
  - 사원별로 적용 급여유형을 지정
  - 급여명세서/계산 시 해당 급여유형의 조합대로 자동 계산

---
